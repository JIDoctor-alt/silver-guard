> 以下决策由项目负责人于评审前已锁定，议题清单中对应条目已预填结论，评审会上无需重复讨论，但需向全体参会人公示确认。

| # | 决策项 | 确认结论 | 决策依据 |
| --- | --- | --- | --- |
| ✅ | **架构模式** | 单体架构（Phase 1），Phase 2 按需拆分微服务 | 降低 MVP 复杂度，加快交付速度 |
| ✅ | **进程间通信** | Dubbo（二进制高效 RPC） | 团队已有经验，性能优于 HTTP |
| ✅ | **后端语言** | Java 21 + Spring Boot 3 | 团队技术栈，技能成熟 |
| ✅ | **ORM** | MyBatis Flex | 团队已有实践，支持 Lambda 查询 |
| ✅ | **AI 框架** | LangChain4j + LangGraph4j + Spring AI Alibaba | 团队技术栈，支持国产 LLM |
| ✅ | **缓存** | Redis（分布式）+ Caffeine（本地）多级缓存 | 团队已有方案，满足性能需求 |
| ✅ | **前端** | Vue 3 + TypeScript + Ant Design Vue | 团队已有实践 |
| ✅ | **家属端** | 微信小程序 | 轻量化，用户无需下载 |
| ✅ | **LLM 选型** | 通义千问（境内合规，成本可控） | 一期暂定，二期支持多模型切换 |
| ✅ | **数据库** | MySQL（主业务）+ ClickHouse（事件时序） | MySQL 团队熟悉，ClickHouse 处理大事件量 |
| ✅ | **容器化** | Docker Compose（Phase 1），K8s 待 Phase 2 | 降低运维复杂度 |
| ✅ | **API 版本策略** | `/api/v1/` 前缀，重大变更升 v2 | RESTful 规范，清晰可控 |

以上决策如有异议，请在评审会前 1 个工作日联系产品/技术负责人。
