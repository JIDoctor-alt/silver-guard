## 🔑 评审前必须对齐的 3 个关键决策

> 以下 3 条是 PRD 中标注为 Open Questions 的，评审会上必须当场拍板，请提前思考立场

### OQ-01：是否与民政/卫健系统打通？
- **选项 A**：一期打通（增加接口开发工作量，影响 Phase 1 上线时间）
- **选项 B**：二期打通（Phase 1 先跑起来）
- **选项 C**：不做（长期不需要）

**建议立场**（供讨论参考）：选项 B（二期打通），理由：一期聚焦核心预警能力，系统稳定后再扩展

---

### OQ-02：家属端形态选什么？
- **选项 A**：微信小程序（推荐，用户无下载门槛，微信生态成熟）
- **选项 B**：独立 App（iOS + Android，需投入更多开发资源）
- **选项 C**：暂不做家属端（Phase 1 网格员优先）

**建议立场**：选项 A（微信小程序），理由：覆盖率高，无需下载，支持微信推送通知

---

### OQ-03：是否需要 110/120 一键联动？
- **选项 A**：一期支持（涉及审批流程，复杂度高）
- **选项 B**：二期支持（系统成熟后推进）
- **选项 C**：不做（涉及医疗/公安专线，成本高）

**建议立场**：选项 B（二期），理由：需要与 110/120 机构单独签订协议，一期集中精力做预警能力

## ❓ 常见疑问 FAQ

### Q1：我在评审中负责什么角色？
| 角色 | 在评审中负责什么 |
| --- | --- |
| 产品负责人 | 主持 PRD 评审，对齐需求范围、优先级、Open Questions |
| 技术负责人 | 主持技术评审，宣布预锁定决策，引导架构讨论 |
| 法务/合规代表 | 重点参与合规与隐私模块（R-14~R-17） |
| 运营/社区代表 | 重点参与试点与运营模块（R-18~R-20） |
| 全体参会人 | 审阅材料 → 在议题讨论中发言 → 确认结论 |

### Q2：如果我不同意某条议题结论怎么办？
- 在讨论环节举手发言，说明立场和理由
- 主持人记录为"反对意见"，但最终按多数意见决定
- 如果认为该议题影响巨大，可要求记为"需再次评审"

### Q3：评审通过后还能改需求吗？
- 可以，但必须走《需求变更流程》（变更申请 → 评审 → 签字 → 更新 PRD 版本）
- 变更影响范围越大（涉及 SLA / 合规 / 架构），评审层级越高

### Q4：遗留事项（Action Items）不完成会怎样？
- 遗留事项是评审中确认需要跟进但未能在会上解决的问题
- 所有遗留事项必须在截止日期前完成，否则影响后续里程碑
- 跟踪方式：录入项目管理工具（飞书任务 / Jira / Teambition）
