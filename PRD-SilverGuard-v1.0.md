## 一、文档说明

### 1.1 目的

本文档为「乐龄守护」智慧养老综合服务平台的需求规格说明书，面向研发与运营团队，清晰传达业务目标、使用场景、功能需求、数据模型、API 接口、非功能需求及验收标准。

### 1.2 范围

- **包含**：IoT 设备感知、AI 异常预警、网格员巡检、家属关怀、健康档案、中医体质、节气养生、知识库、反诈防骗、养老政策、AI 智能问答、国风音乐创作、银龄广场、经典阅读、高德地图集成、积分系统
- **暂不包含**：线下硬件采购与部署、医保/民政系统对接、医疗诊断/处方开具

### 1.3 术语表

| 术语 | 解释 |
|------|------|
| 独居老人 | 长期独自居住、无子女或子女不在身边的 60+ 岁居民 |
| 网格员 | 社区/街道负责基层治理、日常巡检与关怀的一线工作人员 |
| 端边云 | 终端感知 → 边缘计算 → 云平台 → 大模型四级架构 |
| RAG | 检索增强生成，结合知识库的 LLM 问答 |
| SSE | 服务器推送事件，用于 AI 流式回复 |
| LLM | 大语言模型，支持 OpenAI 兼容接口（通义千问/DeepSeek/智谱） |
| 五声音阶 | 宫商角徵羽，中国传统音乐基础音阶 |

## 二、项目概述

### 2.1 背景

中国 60 岁以上老年人口超 **2.8 亿**，独居老人超 **2800 万**。社区网格员人均对接 200+ 户，存在"看不全、到不了、响应慢"的结构性缺口。同时，老年人精神文化需求日益增长——国风音乐、经典阅读、广场舞、中医养生已成为银发群体的核心兴趣点。

**乐龄守护** 从"安全守护"升级为"安全 + 健康 + 文化 + 陪伴"四位一体的智慧养老平台。

### 2.2 核心目标

| 指标 | 目标值 |
|------|--------|
| 异常事件响应时间 | ≤ 30 秒 |
| AI 研判准确率 | ≥ 95% |
| 知识库覆盖领域 | 12 大类 |
| 系统可用性 | ≥ 99.5% |
| 7×24 全天候守护 | 是 |

### 2.3 产品定位

```
乐龄守护 = 安全预警 + 健康管理 + 文化陪伴 + 社区互动
```

### 5.2 模块详细需求

#### M-01 驾驶舱（Dashboard）

- **5 行数据看板**：核心指标卡片、质量指标、设备在线率/误报率/处理完成率（环形图）、系统架构概览 + 最近预警、老人风险等级分布 + 设备列表
- 实时统计：守护老人数、IoT 设备数、今日预警、平均响应时间、累计巡检、AI 准确率、24h 处理率、安全天数
- 风险等级分布：L1 低风险 / L2 中风险 / L3 高风险（进度条展示）
- 端边云架构概览：终端/边缘/云端/AI 四层小型卡片

#### M-02 老人档案（Elder）

- 老人信息：姓名、身份证号（脱敏）、性别、出生日期、地址、风险等级、标签
- 紧急联系人管理
- 风险等级：L1 低风险 / L2 中风险 / L3 高风险

#### M-03 预警事件（Event）

- 事件类型：跌倒、长时间静止、夜间离床、门窗异常、SOS 主动呼叫
- 风险等级：L1 提示 / L2 关注 / L3 紧急 / L4 特急
- 状态流转：OPEN → ASSIGNED → CLOSED / FALSE_ALARM
- 分级筛选、时间排序

#### M-04 设备管理（Device）

- 设备类型：RADAR / IR / DOOR / SMOKE / SOS / BAND
- 设备状态：ONLINE / OFFLINE / FAULT
- 设备列表、在线/离线统计

#### M-05 巡检记录（Patrol）

- 巡检打卡、记录回溯
- 按时间、老人筛选

#### M-06 健康档案（Health）

- **健康数据**：血压（收缩/舒张）、血糖、心率、血氧、体温、睡眠时长、步数、体重
- **摘要卡片**：6 个 Statistic 卡片（平均血压、血糖、心率、睡眠、步数、总记录数）
- **趋势图表**：纯 CSS 柱状图，血压/血糖/心率/睡眠四项趋势
- **记录表格**：日期、血压、血糖、心率、血氧、睡眠、步数、心情彩色标签、来源
- **新增记录**：表单含日期选择器、血压、血糖、心率、血氧、睡眠、步数、心情等字段
- 数据来源：DEVICE / MANUAL / HOSPITAL

#### M-07 中医体质（Constitution）

- **体质评估 Tab**：
  - 9 种体质类型：平和质、气虚质、阳虚质、阴虚质、痰湿质、湿热质、血瘀质、气郁质、特禀质
  - 评估历史表格（日期、体质类型、评分、特征、建议、评估人）
  - 开始评估 Modal（体质类型选择、评分、特征/建议输入）
- **节气养生 Tab**：
  - 当前节气信息卡片（饮食、运动、穴位、起居、食谱）
  - 24 节气卡片网格，点击查看详情
  - 24 节气完整种子数据（立春→大寒，每条含饮食/运动/穴位/起居/食谱）

#### M-08 智能问答（AI Chat）

- **SSE 流式对话**：实时流式响应，打字机效果
- **RAG 知识库**：27 条知识，涵盖 12 个分类
  - 健康管理、中医养生、慢病管理、健康档案、智能设备、养老政策、社区服务、心理关怀、安全知识、运动健身、国风音乐、经典阅读
- **LLM 大模型**：OpenAI 兼容接口（通义千问/DeepSeek/智谱），支持系统提示词配置
- 对话历史记录

#### M-09 音乐陪伴（Music）

- **AI 作词**：主题 + 风格 + 字数 + 押韵，LLM 大模型生成，支持系统提示词配置
- **AI 作曲**：调式 + 音阶 + 速度 + 乐器，LLM 大模型生成，支持系统提示词配置
- **国风乐器知识库**：10 种民族乐器（二胡、笛子、琵琶、古筝、古琴、箫、唢呐、扬琴、阮、马头琴）
- **音乐素材库**：五声音阶、民族调式、节奏型、和声进行、旋律创作、编曲理论、国风美学、戏曲音乐

#### M-10 银龄广场（Square）

- **广场舞 Tab**：30+ 首经典广场舞歌曲，按节奏/流派分类，步法教学
- **社区活动 Tab**：活动创建、报名、日历视图
- **音乐创作 Tab**：音乐知识库素材浏览
- **创作广场 Tab**：
  - 用户创作列表（歌曲/作词/舞步），类型标签
  - **点赞按钮**：心形图标，点击切换点赞/取消，实时计数
  - **评论区域**：发表评论、回复评论、评论列表 + 子回复
  - 通用点赞系统：支持 CREATION / ACTIVITY / DANCE_STEP / SONG 四种目标类型

#### M-11 经典阅读（Reading）

- **书籍分类**：经典 / 诗词 / 哲学 / 养生
- **预置书籍**：10 本（论语、道德经、诗经、唐诗三百首、三字经、弟子规、菜根谭、孙子兵法、黄帝内经·素问、千字文）
- **预置章节**：48 章精选内容（原文 + 白话译文 + 赏析注释）
- **阅读视图**：左侧章节列表 + 右侧正文（大字体居中）
- **字号切换**：小(16px) / 中(20px) / 大(24px) 三档，方便老年人阅读
- **阅读进度**：自动保存，继续阅读按钮直达上次位置
- **书签管理**：收藏章节，右侧抽屉查看，点击跳转
- **章节导航**：上一章/下一章大按钮

#### M-12 知识库（KnowledgeBase）

- **反诈防骗 Tab**：
  - 分类筛选：全部/电话/网络/上门/保健品/投资
  - 8 条预置案例（含风险等级、警示特征、防范措施）
  - 详情 Modal
- **养老政策 Tab**：
  - 分类筛选：全部/养老/医疗/补贴/护理/住房
  - 8 条预置政策（含摘要、详情、适用地区、关键词）
  - 详情 Modal

#### M-13 端边云架构（Architecture）

- 四层架构可视化：
  - 终端感知层（蓝色）：健康监测、环境感知、行为采集、数据传输
  - 边缘计算层（绿色）：实时推理、异常检测、数据清洗、离线自治
  - 云端平台层（紫色）：数据存储、业务中台、数据分析、安全管控
  - 大模型中枢（橙色）：智能对话、RAG 检索、知识图谱、个性化推荐
- 层间纯 CSS 箭头连接
- 底部汇总卡片：数据流向、响应延迟、技术栈、AI 能力

#### M-14 高德地图（Amap）

- 天气查询：当前天气 + 未来预报
- POI 搜索：周边养老设施、医院、药店、超市
- 路径规划：步行/驾车/公交/骑行四种方式
- 地理编码：地址 ↔ 坐标互转
- 养老场景专用：一键查找周边养老机构、医院、公园

## 九、非功能需求

### 9.1 性能

- 预警响应：L3/L4 从数据上报到通知送达 ≤ 30 秒
- 驾驶舱首页：加载 ≤ 2 秒
- 列表分页：1000 条数据响应 ≤ 500ms
- 支持单部署单元 ≥ 5000 户老人并发

### 9.2 可用性

- 全年可用性 ≥ 99.5%
- 关键链路具备主备/降级方案
- 每日自动备份，异地留存 ≥ 30 天

### 9.3 安全与隐私

- 身份证号等 PII 使用不可逆哈希
- 所有对外接口走 HTTPS
- 敏感操作进审计日志，不可篡改
- 符合《个人信息保护法》《数据安全法》

### 9.4 可观测性

- 健康检查接口：`/actuator/health`
- 日志：统一格式，关键操作全链路追踪

### 9.5 老年人体验

- 经典阅读：大字体模式（16px/20px/24px 三档）
- 界面：大按钮、高对比、清晰标签
- 交互：减少操作步骤，信息层级清晰
